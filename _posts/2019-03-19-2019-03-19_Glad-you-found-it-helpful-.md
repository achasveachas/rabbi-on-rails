<p name="0efa" id="0efa" class="graf graf--p graf-after--p">Unfortunately, it’s been almost 3 years since I worked on that project so I don’t remember much of the technical decisions that went into it (to the extent that I understood them back then 😂).</p>
<p name="99d1" id="99d1" class="graf graf--p graf-after--p">Looking at my code it seems like <code class="markup--code markup--p-code">Human</code> and <code class="markup--code markup--p-code">Computer</code> aren’t modules, instead they are both classes that inherit from the <code class="markup--code markup--p-code">Player</code> class. Maybe things changed?</p>
<p name="6efd" id="6efd" class="graf graf--p graf-after--p graf--trailing">In any case, my Ruby is a bit rusty having not used it in a while. If you would like a great book that explains how these things work I highly recommend <a href="https://medium.com/u/b49fae4639c2" data-href="https://medium.com/u/b49fae4639c2" data-anchor-type="2" data-user-id="b49fae4639c2" data-action-value="b49fae4639c2" data-action="show-user-card" data-action-type="hover" class="markup--user markup--p-user" target="_blank">Sandi Metz</a>’s Practical Object Oriented Design In Ruby (affectionately known as POODR in the Ruby community). It’s practically the bible on how to write clean structured Ruby code, and will answer all your questions on what classes and modules are, when do you use them and how plus a lot more.</p>
