Unfortunately, it’s been almost 3 years since I worked on that project so I don’t remember much of the technical decisions that went into it (to the extent that I understood them back then 😂).

Looking at my code it seems like `Human` and `Computer` aren’t modules, instead they are both classes that inherit from the `Player` class. Maybe things changed?

In any case, my Ruby is a bit rusty having not used it in a while. If you would like a great book that explains how these things work I highly recommend [Sandi Metz](https://medium.com/u/b49fae4639c2)’s Practical Object Oriented Design In Ruby (affectionately known as POODR in the Ruby community). It’s practically the bible on how to write clean structured Ruby code, and will answer all your questions on what classes and modules are, when do you use them and how plus a lot more.
