<h3 name="eae5" id="eae5" class="graf graf--h3 graf--leading graf--title">Scratching An Itch</h3>
<p name="c4c7" id="c4c7" class="graf graf--p graf-after--h3">Someone recently posted a question on Quora, the question evoked lots of pleasant memories from my coding trip, so I sat and wrote the following answer, cross-posted here for your reading pleasure.</p>
<p name="317d" id="317d" class="graf graf--p graf-after--p">The question went as follows:</p>
<h4 name="6f31" id="6f31" class="graf graf--h4 graf-after--p"><a href="https://www.quora.com/Have-you-ever-built-a-coding-project-from-scratch-If-so-what-did-you-build" data-href="https://www.quora.com/Have-you-ever-built-a-coding-project-from-scratch-If-so-what-did-you-build" class="markup--anchor markup--h4-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--h4-strong">Have you ever built a coding project from scratch? If so, what did you build?</strong></a></h4>
<p name="f630" id="f630" class="graf graf--p graf-after--h4">Others are nitpicking on the “from scratch” part. But as Carl Sagan famously said, “If you wish to make an apple pie from scratch, you must first invent the universe.”</p>
<p name="66e6" id="66e6" class="graf graf--p graf-after--p">For the purpose of this answer, I will define “from scratch” to mean taking an idea and fleshing it out into a working project, using all the tools at your disposal.</p>
<p name="bf45" id="bf45" class="graf graf--p graf-after--p">As an “almost graduate” from <a href="https://medium.com/u/973c5cbfb09b" data-href="https://medium.com/u/973c5cbfb09b" data-anchor-type="2" data-user-id="973c5cbfb09b" data-action-value="973c5cbfb09b" data-action="show-user-card" data-action-type="hover" class="markup--user markup--p-user" target="_blank">Flatiron School</a>, I have a few projects I made. The curriculum is built in such a way that as you progress through the courses you build “freestyle” projects using the knowledge you just learned.</p>
<p name="cb2c" id="cb2c" class="graf graf--p graf-after--p">Here are some of the ones I really enjoyed making:</p>
<h4 name="1cc3" id="1cc3" class="graf graf--h4 graf-after--p"><strong class="markup--strong markup--h4-strong">Tic Tac Toe with a working AI</strong></h4>
<p name="81e7" id="81e7" class="graf graf--p graf-after--h4">The first project they had us build was a functioning CLI Tic Tac Toe game built using Ruby. This project is not fully “from scratch” being that Flatiron provided the file structure, and a lot of the code was copy/pasted from earlier lessons in the course. Nevertheless, the AI player was completely us (well, not so much an AI as a bunch of <code class="markup--code markup--p-code">elsif</code> statements, but it was still fun to code), and I had a lot of fun personalizing the game and giving it personality. I still consider it one of the funnest projects I did.</p>
<p name="b76c" id="b76c" class="graf graf--p graf-after--p">You can find the code for it <a href="https://github.com/achasveachas/Tic-Tac-Toe-with-AI" data-href="https://github.com/achasveachas/Tic-Tac-Toe-with-AI" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a> (including a readme with instructions on how to run it), and a blog post I wrote about coding it <a href="https://blog.yechiel.me/tic-tac-tips-517d5c80f47d" data-href="https://blog.yechiel.me/tic-tac-tips-517d5c80f47d" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Tic Tac Tips</a>.</p>
<h4 name="fd9b" id="fd9b" class="graf graf--h4 graf-after--p">DansDeals</h4>
<p name="ec0a" id="ec0a" class="graf graf--p graf-after--h4">The first project that was <strong class="markup--strong markup--p-strong">really </strong>from scratch was at the end of the OO Ruby section of the curriculum. The requirements of the project were that we make a CLI Ruby gem that scrapes a website or API for data that a user can interact with through the command line.</p>
<p name="a191" id="a191" class="graf graf--p graf-after--p">I chose to scrape a deals site called DansDeals (hence the name of the gem), and surprisingly the gem still works as of the writing of this answer (that says less to my skills as a developer than it does to the fact that that website hasn’t been updated since I deployed the gem).</p>
<p name="123b" id="123b" class="graf graf--p graf-after--p">This gem has been deployed to <a href="https://rubygems.org/gems/dansdeals" data-href="https://rubygems.org/gems/dansdeals" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">RubyGems</a>, and anyone can play with it (assuming you have Bundler) by running <code class="markup--code markup--p-code">gem install dansdeals</code> in your command line and then running <code class="markup--code markup--p-code">dansdeals</code>.</p>
<p name="5551" id="5551" class="graf graf--p graf-after--p">You can find the code for this project <a href="https://github.com/achasveachas/dansdeals" data-href="https://github.com/achasveachas/dansdeals" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a>, and my blog post <a href="https://blog.yechiel.me/gem-install-swim-82ff7a95e605" data-href="https://blog.yechiel.me/gem-install-swim-82ff7a95e605" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">gem install swim</a></p>
<h4 name="e15e" id="e15e" class="graf graf--h4 graf-after--p">Freddit</h4>
<p name="c0f7" id="c0f7" class="graf graf--p graf-after--h4">The next project came when we finished the section covering <a href="https://en.wikipedia.org/wiki/Sinatra_%28software%29" data-href="https://en.wikipedia.org/wiki/Sinatra_(software)" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Sinatra</a>. For this project, I chose to make a small messaging board called Freddit (no relationship to Reddit of course).</p>
<p name="e05a" id="e05a" class="graf graf--p graf-after--p">This was a fairly fun project to code, it included more advanced concepts, like User authentication and validation, it featured admin users who could block or ban other users. Most importantly, regarding the question at least, it was even more “from scratch” than other projects I made being that the framework I used to build it, <a href="http://www.rubydoc.info/gems/corneal/1.2.0" data-href="http://www.rubydoc.info/gems/corneal/1.2.0" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Corneal</a>, is an open-source framework that I contributed to and <a href="https://blog.yechiel.me/of-scaffolds-and-gems-140bdbe2e005" data-href="https://blog.yechiel.me/of-scaffolds-and-gems-140bdbe2e005" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">helped build</a>!</p>
<p name="a762" id="a762" class="graf graf--p graf-after--p">You can find the code for Freddit <a href="https://github.com/achasveachas/freddit" data-href="https://github.com/achasveachas/freddit" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a>, and my blog post <a href="https://blog.yechiel.me/i-challenge-you-sinatra-c6f875e29db7" data-href="https://blog.yechiel.me/i-challenge-you-sinatra-c6f875e29db7" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">I Challenge You Sinatra</a></p>
<h4 name="e3ce" id="e3ce" class="graf graf--h4 graf-after--p">FlyLog</h4>
<p name="4a61" id="4a61" class="graf graf--p graf-after--h4">Next came Rails, and when it was time for the project I decided to chose something I was passionate about, flying!</p>
<p name="36dd" id="36dd" class="graf graf--p graf-after--p">FlyLog is a pilot’s logbook built entirely with Rails, where pilots could log their hours, keep count of the planes they’ve flown, and see the total number of flight hours they have.</p>
<p name="ce34" id="ce34" class="graf graf--p graf-after--p">This project actually made it all the way to deployment, you can find it at <a href="https://flylogger.herokuapp.com/" data-href="https://flylogger.herokuapp.com/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">FlyLogger.herokuapp.com</a></p>
<p name="dd91" id="dd91" class="graf graf--p graf-after--p">You can find the code for it <a href="https://github.com/achasveachas/FlyLog" data-href="https://github.com/achasveachas/FlyLog" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a>, and my blog post <a href="https://blog.yechiel.me/fly-off-the-rails-78f3e4e82e72" data-href="https://blog.yechiel.me/fly-off-the-rails-78f3e4e82e72" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Fly Off The Rails</a></p>
<h4 name="674e" id="674e" class="graf graf--h4 graf-after--p">Freddit Revisited</h4>
<p name="cabd" id="cabd" class="graf graf--p graf-after--h4">After Rails, we switched to JavaScript and some basic libraries like jQuery. The next challenge was to create a Rails app that uses JavaScript AJAX calls to render content on the page. The instructors suggested we just add JavaScript to our Rails project, but I decided to do something a bit more challenging, and rewrite Freddit in Rails.</p>
<p name="0c32" id="0c32" class="graf graf--p graf-after--p">In the process I added more features, such as the ability for readers to reply directly to comments, nesting the comments and their replies, a bit closer to its namesake.</p>
<p name="0a97" id="0a97" class="graf graf--p graf-after--p">This project was also deployed on Heroku, you can see it at <a href="https://freddit-jq.herokuapp.com/" data-href="https://freddit-jq.herokuapp.com/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Freddit-JQ.herokuapp.com</a></p>
<p name="dca8" id="dca8" class="graf graf--p graf-after--p">The code is on GitHub <a href="https://github.com/achasveachas/freddit-jq" data-href="https://github.com/achasveachas/freddit-jq" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here,</a> and my blog post <a href="https://blog.yechiel.me/challenges-revisited-33ae6ab648ae" data-href="https://blog.yechiel.me/challenges-revisited-33ae6ab648ae" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Challenges Revisited</a></p>
<h4 name="8c4b" id="8c4b" class="graf graf--h4 graf-after--p">App-Tracker</h4>
<p name="67d3" id="67d3" class="graf graf--p graf-after--h4">Finally, we went on to more advanced frameworks like React, and then came the time for the final project. The requirements for the final project were that we use a Rails REST API for the back-end, and use React for the client side application.</p>
<p name="62b3" id="62b3" class="graf graf--p graf-after--p">My idea for this project was to build an app to let me keep track of all the job applications I’d be submitting after I’m finished with Flatiron, I am currently in middle of the application so I won’t post links to it yet, I’ll wait until it’s finished and then update the answer (well actually, it shouldn’t be too hard to find on my GitHub page, it’s my latest busy repo :)</p>
<blockquote name="4053" id="4053" class="graf graf--blockquote graf-after--p">Edit: The project is mostly finished now, you can find it at <a href="https://app-tracker-react.herokuapp.com/" data-href="https://app-tracker-react.herokuapp.com/" class="markup--anchor markup--blockquote-anchor" rel="noopener" target="_blank">app-tracker-react.herokuapp.com,</a> the GitHub repo is <a href="https://github.com/achasveachas/app-tracker-react" data-href="https://github.com/achasveachas/app-tracker-react" class="markup--anchor markup--blockquote-anchor" rel="noopener" target="_blank">here</a>.</blockquote>
<p name="7502" id="7502" class="graf graf--p graf-after--blockquote">So those are some of the projects I enjoyed making. You can read more about my coding on my blog <a href="https://blog.yechiel.me/" data-href="https://blog.yechiel.me/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Coding Hassid</a></p>
<p name="7299" id="7299" class="graf graf--p graf-after--p graf--trailing">Happy coding!</p>
