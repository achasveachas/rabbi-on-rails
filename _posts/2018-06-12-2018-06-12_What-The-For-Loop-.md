<h3 name="d3bb" id="d3bb" class="graf graf--h3 graf--leading graf--title">What The For Loop?</h3>
<h4 name="bcc3" id="bcc3" class="graf graf--h4 graf-after--h3 graf--subtitle">Learning From A Popular Interview Question</h4>
<p name="15d5" id="15d5" class="graf graf--p graf-after--h4">This is another post in the genre of technical interview questions I’ve come across during my job hunt, and how to go about solving them (For more posts see <a href="https://blog.yechiel.me/get-your-fizz-buzz-on-543f2a327a9d" data-href="https://blog.yechiel.me/get-your-fizz-buzz-on-543f2a327a9d" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a>, <a href="https://blog.yechiel.me/parenthetically-speaking-a275f86c833e" data-href="https://blog.yechiel.me/parenthetically-speaking-a275f86c833e" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a>, and <a href="https://blog.yechiel.me/parenthetically-speaking-2-db6386eb9db3" data-href="https://blog.yechiel.me/parenthetically-speaking-2-db6386eb9db3" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a>).</p>
<figure name="7e04" id="7e04" class="graf graf--figure graf-after--p"><img class="graf-image" data-image-id="0*YARX8X7V6XsQNSMe" data-width="960" data-height="640" src="/assets/images/posts/2018-06-12-2018-06-12_What-The-For-Loop--0.jpg"></figure><p name="77ed" id="77ed" class="graf graf--p graf-after--figure">Today’s question is a simple one, but the correct answer gets to the basics of a very popular feature in most languages; so popular, we almost never give it any thought.</p>
<p name="05d7" id="05d7" class="graf graf--p graf-after--p">The question is the following: write a function that prints all the odd numbers up to 20.</p>
<p name="a58f" id="a58f" class="graf graf--p graf-after--p">Sounds simple, right? Any beginner will tell you that this is a job for a <code class="markup--code markup--p-code">for</code> loop. So let’s write it out:</p>
<pre name="c173" id="c173" class="graf graf--pre graf-after--p">for(let i=0; i&lt;=20; i++){<br> if(i%2===1){<br> console.log(i)<br> }<br>} </pre>
<p name="6b6d" id="6b6d" class="graf graf--p graf-after--pre">In (over)simplified English: we run a loop for 20 iterations, during each iteration of the loop we check to see if the value of a variable <code class="markup--code markup--p-code">i</code> is odd and if it is we print it to the console.</p>
<p name="bd13" id="bd13" class="graf graf--p graf-after--p">The above function satisfies the requirements. If you run it in your console, you will see it does the job. The question is, is there a better way to do things?</p>
<h4 name="e3e9" id="e3e9" class="graf graf--h4 graf-after--p">What The For Loop?</h4>
<p name="c446" id="c446" class="graf graf--p graf-after--h4"><em class="markup--em markup--p-em">Note: For more on for loops in JavaScript see the </em><a href="https://www.w3schools.com/js/js_loop_for.asp" data-href="https://www.w3schools.com/js/js_loop_for.asp" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><em class="markup--em markup--p-em">documentation on W3schools</em></a></p>
<p name="ea29" id="ea29" class="graf graf--p graf-after--p">Obviously we can (otherwise there would be no point to this blog post); to understand how, let’s take a closer look at the syntax of the <code class="markup--code markup--p-code">for</code> loop, and more specifically, the first line:</p>
<p name="b0ae" id="b0ae" class="graf graf--p graf-after--p"><code class="markup--code markup--p-code">for(let i=0; i&lt;20; i++){</code></p>
<p name="f1c7" id="f1c7" class="graf graf--p graf-after--p">We can see that the for loop takes as an argument three statements separated by <code class="markup--code markup--p-code">;</code>’s. Let’s take a closer look at them.</p>
<p name="c2ad" id="c2ad" class="graf graf--p graf-after--p">The first statement is an expression that is run only one time, before the <code class="markup--code markup--p-code">for</code> loop gets executed. It’s usually used to initialize a counter, but you can put any valid JavaScript expression, or none at all (for example, if you already initialized your counter outside of the <code class="markup--code markup--p-code">for</code> loop). In the example above, the first statement defines a variable <code class="markup--code markup--p-code">i</code> and sets its value to <code class="markup--code markup--p-code">0</code>.</p>
<p name="9581" id="9581" class="graf graf--p graf-after--p">The second statement is a conditional which gets evaluated before each iteration of the loop. As long as the conditional evaluates to <code class="markup--code markup--p-code">true</code> the loop keeps running. Once conditions change so that the second statement evaluates to <code class="markup--code markup--p-code">false</code>, we break out of the loop. In our example, the conditional is <code class="markup--code markup--p-code">i &lt; 20</code> so the loop runs as long as the value of <code class="markup--code markup--p-code">i</code> remains below 20.</p>
<figure name="6910" id="6910" class="graf graf--figure graf--iframe graf-after--p"><iframe src="https://giphy.com/embed/MJ6SslGZEYKhG/twitter/iframe" width="435" height="320" frameborder="0" scrolling="no"></iframe><figcaption class="imageCaption">Infinite loops are infinite loops</figcaption></figure><p name="b9ac" id="b9ac" class="graf graf--p graf-after--figure">The third statement is another expression. This expression is run <strong class="markup--strong markup--p-strong">after </strong>every iteration of the loop. It’s usually used to increment the counter, but again, you can put any legal JavaScript in there, and it will run (of course, if you aren’t using it to increment a counter you need to make sure you have another way of changing your conditional in the second statement to true, otherwise you will be stuck with a dreaded infinite loop). In our previous example, we are using the third statement to increment the value of <code class="markup--code markup--p-code">i</code> after each iteration, so that after 20 iterations <code class="markup--code markup--p-code">i</code> is equal to <code class="markup--code markup--p-code">20</code>, <code class="markup--code markup--p-code">i &lt; 20</code> evaluates to <code class="markup--code markup--p-code">true</code>, and we break out of the loop.</p>
<h4 name="430d" id="430d" class="graf graf--h4 graf-after--p">We Can Do <strong class="markup--strong markup--h4-strong">Better</strong>
</h4>
<p name="ea34" id="ea34" class="graf graf--p graf-after--h4">Now let’s take a look at our function and see how we can optimize it.</p>
<p name="62e0" id="62e0" class="graf graf--p graf-after--p">As a refresher here is the function:</p>
<pre name="9aca" id="9aca" class="graf graf--pre graf-after--p">for(let i=0; i&lt;=20; i++){<br> if(i%2===1){<br> console.log(i)<br> }<br>}</pre>
<p name="0f0d" id="0f0d" class="graf graf--p graf-after--pre">So we set the value of <code class="markup--code markup--p-code">i</code> to zero and start the loop. At each iteration of the loop we check the current value of <code class="markup--code markup--p-code">i</code>, if it’s odd, we log it to the console, and then we increment <code class="markup--code markup--p-code">i</code> by 1 and rerun the loop until <code class="markup--code markup--p-code">i</code> hits 20 at which point we break the loop.</p>
<p name="ad1d" id="ad1d" class="graf graf--p graf-after--p">How can we optimize this?</p>
<p name="f940" id="f940" class="graf graf--p graf-after--p">The key is in the third statement. As mentioned earlier, nothing is forcing us to increment our counter by 1 in the third statement; we can do whatever we want. Combining that knowledge with the fact that 1 is an odd number, and that adding 2 to an odd number gives us an odd number as well and the result is a loop that only has to run half of the iterations our previous attempt used.</p>
<p name="4f0b" id="4f0b" class="graf graf--p graf-after--p">Try putting the following in your console and see how it runs the same:</p>
<pre name="edb7" id="edb7" class="graf graf--pre graf-after--p">for(let i=1; i&lt;=20; i +=2 ){<br> console.log(i)<br>}</pre>
<p name="19c0" id="19c0" class="graf graf--p graf-after--pre">The differences between this function and the previous one is that here we set the initial value of <code class="markup--code markup--p-code">i</code> to 1 and instead of incrementing <code class="markup--code markup--p-code">i</code> by one for each iteration we increment it by two (we also got rid of the <code class="markup--code markup--p-code">if</code> statement because we know that now <code class="markup--code markup--p-code">i</code> is always odd, so we just log the value of <code class="markup--code markup--p-code">i</code> each time without checking).</p>
<p name="acc7" id="acc7" class="graf graf--p graf-after--p">So we see how sometimes knowing how things work under the hood can help us when we want to tweak them beyond the way they are usually used.</p>
<p name="9b35" id="9b35" class="graf graf--p graf-after--p">I hope this post inspired you to sometimes delve a little deeper, even into concepts that “everyone knows”.</p>
<p name="82cd" id="82cd" class="graf graf--p graf-after--p graf--trailing">Happy coding!</p>
