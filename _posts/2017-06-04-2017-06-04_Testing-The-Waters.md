<h3 name="c159" id="c159" class="graf graf--h3 graf--leading graf--title">Testing The Waters</h3>
<h4 name="290a" id="290a" class="graf graf--h4 graf-after--h3 graf--subtitle">Life After The Bootcamp</h4>
<p name="9468" id="9468" class="graf graf--p graf-after--h4">As mentioned in my <a href="https://blog.yechiel.me/reactjs-app-with-rails-api-4ffb12ba6608" data-href="https://blog.yechiel.me/reactjs-app-with-rails-api-4ffb12ba6608" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">latest post</a>, I recently graduated <a href="https://medium.com/u/973c5cbfb09b" data-href="https://medium.com/u/973c5cbfb09b" data-anchor-type="2" data-user-id="973c5cbfb09b" data-action-value="973c5cbfb09b" data-action="show-user-card" data-action-type="hover" class="markup--user markup--p-user" target="_blank">Flatiron School</a>’s Web Developer bootcamp. After the celebrations calmed down a bit, the question that now begs an answer is “now what?”</p>
<p name="e41c" id="e41c" class="graf graf--p graf-after--p">While going through the program, the question always came up “what are your plans after graduation?” None of us deluded ourselves into thinking that taking one course would magically transform us into 10X web developers; what Flatiron did do a great job with, was turning us into first class learners, and gosh, there’s so much to learn.</p>
<p name="54dd" id="54dd" class="graf graf--p graf-after--p">Whenever those discussions came up, I always joked that I have a huge to-do list of things I need to learn after graduation. The problem is that it’s a horizontal list; I want to do all of the things first!</p>
<figure name="e766" id="e766" class="graf graf--figure graf-after--p"><img class="graf-image" data-image-id="1*nfM9x8kN_5RPlbZE98seNA.jpeg" data-width="267" data-height="189" src="/assets/images/posts/2017-06-04-2017-06-04_Testing-The-Waters-0.jpeg"><figcaption class="imageCaption">And do them first!</figcaption></figure><p name="2ca2" id="2ca2" class="graf graf--p graf-after--figure">So now that I graduated, all jokes were off, I had to choose what my next focus will be (other than the job hunt for my next developer position of course). Should I focus on brushing up my basic CS skills with algorithms and data structures? Should I go ahead and learn a new language/framework?</p>
<p name="f968" id="f968" class="graf graf--p graf-after--p">In the end, I decided to focus on writing tests.</p>
<h3 name="d5d1" id="d5d1" class="graf graf--h3 graf-after--p">Why tests?</h3>
<p name="6db1" id="6db1" class="graf graf--p graf-after--h3">Let me start by explaining what I mean with tests for my non-technical audience.</p>
<p name="6362" id="6362" class="graf graf--p graf-after--p">Suppose you were making a website. For the sake of this thought experiment, let’s pretend all this website can do is allow users to sign up, log in, log out, and delete their accounts (it’s like Facebook but without the annoying parts).</p>
<p name="9425" id="9425" class="graf graf--p graf-after--p">So let’s say it took me an hour to put that together, I now have to make sure it works. That shouldn’t be too complicated, my site can only do 4 things, so I sign up a test user, log out, log back in with the user I just made, and delete. Perfect, it all works smoothly, under budget and ahead of schedule.</p>
<p name="6b36" id="6b36" class="graf graf--p graf-after--p">Now I want my website to do something else; maybe a user should be able to edit her information. So I add that functionality, and to make sure the new feature works I log in with my test user and try to edit the username. That works as well, but wait, I also want to make sure that while coding the “Edit User” feature I didn’t break anything else, so I go back to my application, sign up a new test user, log out, log back in with an existing user, edit the information, and delete the user. In the process, I realize that somehow a bug crept in that broke the “Delete User” functionality, so I go back into the code to fix it, and in order to verify that everything is fixed, I sign up a new test user, log out, log back in with an existing user, edit the information, and delete the user.</p>
<p name="dc82" id="dc82" class="graf graf--p graf-after--p">It doesn’t take much to realize that this process can get very time-consuming quickly, and it only gets worse as I add more features to my website.</p>
<h3 name="b04c" id="b04c" class="graf graf--h3 graf-after--p">Enter tests</h3>
<p name="5f3d" id="5f3d" class="graf graf--p graf-after--h3">The solution is to automate the process. For every project I make, I would add a test suite. So for my above application, my test suite would contain a script that signs a user up, logs an existing user in, etc.</p>
<p name="cb30" id="cb30" class="graf graf--p graf-after--p">Ideally, I would aim to have a test cover every feature in my application, so that whenever I make any change to the code, all I have to do is run the tests, and within a few seconds I would have confirmation that I didn’t break anything, and if I did, I would know exactly where it was broken so I could fix it.</p>
<p name="3a90" id="3a90" class="graf graf--p graf-after--p">(To my more technical readers, the above is an extreme oversimplification of the functions of tests, but I don’t need to explain that to <strong class="markup--strong markup--p-strong">you</strong>).</p>
<p name="8d66" id="8d66" class="graf graf--p graf-after--p">At Flatiron we dealt with tests a <strong class="markup--strong markup--p-strong">lot</strong>, we read them, ran them, banged our heads when they failed, and celebrated when we got them to pass.</p>
<figure name="6a53" id="6a53" class="graf graf--figure graf-after--p"><img class="graf-image" data-image-id="1*STzY3STu64_HqzD9KiCuOQ.gif" data-width="362" data-height="381" src="/assets/images/posts/2017-06-04-2017-06-04_Testing-The-Waters-1.gif"><figcaption class="imageCaption">This GIF used to be real popular on our Flatiron Slack</figcaption></figure><p name="f1cc" id="f1cc" class="graf graf--p graf-after--figure">In fact, most of the learning was not done through lectures or books, it was done through a pattern I call TDL (Test Driven Learning, loosely based off of TDD, Test Driven Development). We would start by cloning a GitHub repo, running the test suite, analyzing the error messages, and doing what had to be done to get them passing.</p>
<figure name="c509" id="c509" class="graf graf--figure graf-after--p"><img class="graf-image" data-image-id="1*d-UV-tZNlcMc47ScDWGFMQ.png" data-width="851" data-height="409" data-is-featured="true" src="/assets/images/posts/2017-06-04-2017-06-04_Testing-The-Waters-2.png"><figcaption class="imageCaption">Fellow Learners are known to wake up in middle of the night screaming about this lab…</figcaption></figure><p name="c30a" id="c30a" class="graf graf--p graf-after--figure">The one thing we didn’t do much of, was actually <strong class="markup--strong markup--p-strong">writing</strong> tests. And that, I believe, is unfortunate. Writing tests is such an important part of writing software, it would have been nice to spend some time learning how to write them properly.</p>
<p name="8fb2" id="8fb2" class="graf graf--p graf-after--p">In my <a href="https://github.com/achasveachas/app-tracker" data-href="https://github.com/achasveachas/app-tracker" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">last project</a> I did write some tests on my own, but nowhere near as much as I would have liked to. So now that I’m “on my own”, I decided to fix that situation. For the coming week or two, I’ll be reading up on testing patterns, adding tests to my old projects, and maybe even write an entire TDD app from the bottom up.</p>
<p name="aa2d" id="aa2d" class="graf graf--p graf-after--p">Based on some friends’ recommendations I started reading through <a href="https://pragprog.com/book/nrtest2/rails-4-test-prescriptions" data-href="https://pragprog.com/book/nrtest2/rails-4-test-prescriptions" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Rails 4 Test Prescriptions</a>, if anyone reading this has some more recommendations please do leave them in the comments.</p>
<p name="a6bd" id="a6bd" class="graf graf--p graf-after--p graf--trailing">As always, Happy Coding!</p>
